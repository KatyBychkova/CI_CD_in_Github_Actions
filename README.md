# CI/CD in GitHub Actions

## 1. Выбранный проект

В рамках данного задания был реализован простой **Java-проект** с использованием **Maven** и **JUnit 5**, предназначенный для демонстрации работы CI/CD pipeline в GitHub Actions.

Проект включает в себя:
- основной Java-класс с бизнес-логикой;
- модульные тесты (unit tests);
- Maven Wrapper для воспроизводимой и независимой от окружения сборки;
- CI/CD pipeline, настроенный с использованием GitHub Actions.

Проект является **кроссплатформенным** и успешно собирается и тестируется на следующих операционных системах:
- Linux;
- Windows;
- macOS.

---

## 2. Описание настроенных jobs

### 2.1 Job `build-and-test`

Job `build-and-test` отвечает за автоматическую сборку и тестирование проекта.

#### Триггеры запуска
Pipeline запускается автоматически при следующих событиях:
- push в ветку `master`;
- создание pull request в ветку `master`.

#### Основные особенности job
- используется **matrix strategy** для параллельного запуска сборки и тестов на нескольких операционных системах:
  - Linux (`ubuntu-latest`);
  - Windows (`windows-latest`);
  - macOS (`macos-latest`);
- выполняется установка **JDK 17**;
- сборка и тестирование выполняются с использованием **Maven Wrapper**;
- запускается сборка проекта и выполнение unit-тестов;
- добавлен шаг `chmod +x mvnw` для корректной работы Maven Wrapper в средах Linux и macOS.

---

### 2.2 Job `deploy`

Job `deploy` отвечает за этап деплоя приложения.

#### Особенности job
- запускается **только после успешного завершения job `build-and-test`**;
- выполняется в среде Linux (`ubuntu-latest`);
- в рамках учебного задания деплой реализован в виде **демонстрационного шага**, подтверждающего корректное завершение CI/CD pipeline.
